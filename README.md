# 家計簿ダッシュボード（第1段階）

ZaimとMoneyForward MEのCSVを取り込み、月ごとの収支を集計・可視化し、反省を記録するアプリ。
Next.js（App Router / JavaScript）で作成。家計データは外部に送信せず、ローカルの`/data`にJSONで蓄積する。

## セットアップ・起動

```bash
npm install
npm run dev        # http://localhost:3000
```

初期状態で **6月分のサンプルが読み込み済み**（`data/transactions.json`）なので、起動するとすぐダッシュボードが表示されます。
自分のデータで試すときは画面右上の「全消去」→「CSV取込」でZaim/MFのCSVを取り込んでください。

## 使い方（ダッシュボードタブ）

- **対象月ドロップダウン** … 取り込んだデータの月を切り替え
- **CSV取込** … Zaim・MoneyForwardのCSV（Shift-JIS）を選んで取込。ヘッダで自動判別
- **サマリーカード** … 収入 / 支出（投資除く） / 収支 / 貯蓄率
- **支出の内訳円グラフ** … 大分類5つ（食費・生活・娯楽・美容健康・固定費）
- **小分類の内訳表** … 大分類ごとに小分類・件数・金額・構成比
- **反省欄** … 月ごとに保存

## ディレクトリ構成

```
app/
  page.js                     … トップ（BudgetApp を描画）
  layout.js, globals.css
  api/transactions/route.js   … 明細の取得/保存(マージ)/全消去
  api/reflections/route.js    … 月ごとの反省の取得/保存
components/                   … ダッシュボードのUI一式
lib/
  mapping.js    ★カテゴリマッピング（このプロジェクトの核心。ここだけ直せば調整可）
  transform.js  … CSV行 → 取引明細への変換（振替/残高調整の除外、集計設定の反映）
  aggregate.js  … 明細 → 月次集計（収支・貯蓄率・円グラフ・内訳表）
  parseCsv.js   … ブラウザ側でShift-JIS→UTF-8変換してPapa Parse
  storage.js    … /data のJSON読み書き
  format.js     … 表示用フォーマッタ
data/
  transactions.json … 取引明細（1件ずつ）
  reflections.json  … { "2026-06": "..." }
scripts/
  seed.mjs   … 任意の複数CSV(Zaim/MF・複数月)を判別して data/transactions.json に取込
  verify.mjs … data/transactions.json を月ごとに集計してコンソール出力（Excel突き合わせ用）
  seed-june.mjs / verify-june.mjs … 旧・6月専用（seed.mjs / verify.mjs に統合済み）
```

再取込の例（5月・6月をまとめて投入して検証）:
```bash
node scripts/seed.mjs "<Zaim_6月>.csv" "<MF_6月>.csv" "<Zaim_5月>.csv" "<MF_5月>.csv"
node scripts/verify.mjs
```

## 設計上の重要ポイント

### 1. 明細を1件ずつ保存
月合計ではなく取引を1行ずつ保存し、集計は都度計算（指示書の最重要方針）。これにより第2段階の明細/予算/比較タブも同じデータソースから作れます。

### 2. 集計から除外する行
- **振替 / 残高調整**（Zaim `方法`= transfer/balance、MF `振替`=1）… 口座間移動なので取り込まない
- **投資/積立** … 支出に含めず「投資」として別集計
- **Zaimの「集計の設定 = 集計に含めない」の行** … 保存はするが集計から除外（`included:false`）。
  クレジットの一括支払い（振替メルペイ／三井住友カード等）、チャージ、利息など。**MF側の明細と二重計上しないための処理**で、これが6月のExcel値と合わせる鍵になります。指示書のマッピング表には明記されていない列ですが、実データ上ここを尊重しないと数字が大きくズレます。要確認ポイントです。

### 3. 未分類の可視化
どのマッピングにも該当しない支出は「未分類」に落とし、警告バナー＋内訳表で件数・合計を表示します。6月サンプルでは **未分類0件**（全行マッピング成功）。

### 4. 貯蓄率の計算（Excel踏襲）
- 収支 = 収入 − 支出（投資は含めない）
- 暫定貯蓄率 = 収支 ÷ 収入
- 現金貯蓄率 = 暫定貯蓄率 − 投資比率（投資額 ÷ 収入）

## カテゴリ改称への対応（5月サンプル反映）

Zaim側でカテゴリ名を一部改称したため、6月時点の名前で作ったマッピングだと未分類に落ちる箇所がありました。`lib/mapping.js` に新旧どちらの名前も登録済みなので、5月・6月とも未分類0件でマッピングされます。

| 元カテゴリ（新・5月〜） | 旧・6月まで | 大分類/小分類 |
|---|---|---|
| 固定費用 / 固定費・サブスク | 固定費用 / サブスク・月額課金 | 固定費 / 固定サブスク |
| 娯楽 / 本 | 娯楽 / 書籍 | 娯楽 / 本 |
| 固定費用 / サブスク(年払い) | （5月で新規） | 固定費 / 年払いサブスク |
| （MF）趣味・娯楽 / 突発サブスク | （新規） | 娯楽 / 突発サブスク |
| （MF）食費 / 娯楽食費 | （新規） | 食費 / 娯楽食費 |

## 明細タブ（実装済み）
Zaim/MF統合済みの取引を1件ずつ表示します。不要データの点検用に以下を用意:
- フィルタ: 対象月 / 種別(支出・収入・投資) / 大分類 / 小分類 / 集計対象 / キーワード検索
- 「集計対象」を **対象外のみ** にすると、集計から除外している行（クレジット一括払い・チャージ・利息・振替相当など）だけを表示。何を除外しているか一覧で確認できます
- 対象外の行は斜線ハッチングで区別、日付・金額でソート可、フィルタ後の種別別合計を表示
